## Zlatan site + landing page + Meowdoku tactic engine overhaul — Wolfgang (Session 05)

**Playground root (new)**
`index.html` — landing page linking to Meowdoku and Zlatan. `.github/workflows/deploy.yml` — deploy path changed from `meowdoku/` to `./`; the old path meant the landing page and Zlatan were built but never actually served.

**`zlatan/` (new)**
`index.html` + `style.css` + `app.js` — quotes/facts site, black/gold theme. Random featured quote + fact shown per visit. 👍/👎 voting per item, persisted in `localStorage['zlatan_votes']`.

**Meowdoku `app.js` — palette + Apply**
New `X_MARK_MODE`/`CAT_MODE` palette swatches for manual correction (eraser now also clears cat/X on a cell). `markCatEliminations()` — Apply on a cat placement now auto-X's its row/column/color/8-neighbors, not just its row.

**Meowdoku `app.js` — tactic engine (`generateHintText`, `showHint`)**
Rewritten twice. First: expanded from 6 to 11 tactics (added forced-column, vector isolation on region-confinement, Set Saturation K=2..4, Line-Segment Halo, Diagonal Pinch, Conjugate Pair, depth-2 propagation contradiction). Second, larger rewrite: every deduction now returns `{name, rule, here, checked, ...}` instead of one sentence, rendered as three lines (technique name → general rule → this-board application) plus a dimmed "checked first" trail. Added Line Lock (reverse), Crowding (hidden), generalized the old Squeeze into Shared Shadow (any candidate count, not capped at 2–3), and added Team Shadow (two-unit joint reasoning — shipped a same-cell-overlap bug and fixed it before it stuck). What-If (the contradiction tester) redesigned from "batch every dead end into one Apply" to "show the shallowest forced chain one at a time, chain drawn as dashed ghost cats, with an Apply All button for the rest" — `state.pendingAllAction` / `runApplyAll()`.

**Meowdoku `image.js` — screenshot-import fixes**
`approxColorName` now parses `rgb(r,g,b)` strings (previously only `#hex` — every imported color silently named itself "Red"). Cat detection: dense 9×9 near-black sampling (`<60` brightness) replacing a sparse, then briefly too-loose (`<95`, false-positived on dark tiles), detector. Grid-size band threshold 0.4→0.2 (fixes an 8×8 board misread as 10×10 on heavily X-marked boards). X-mark detection: desaturation test (`min(r,g,b)>195 && max−min<30`) replacing a background-relative-brightness test that missed marks on light backgrounds like pink.

**Meowdoku `style.css`**
New swatch styles, `.hint-name/.hint-rule/.hint-here/.hint-checked/.hint-chain-step/.hint-more`, `.cell.hint-ghost` (dashed pulse for What-If's hypothetical chain cats), `.cell.hint-locked` now pulses red instead of a static faint ring, `.btn-apply-all`. Visual polish: `grid-auto-rows: 1fr` (fixed uneven row heights), palette overflow fade (conditional on actual overflow), AI-button restyled quiet, status line muted, X-mark styles unified, remaining German strings translated.

**Docs**
`docs/architecture.md`, `docs/features/coaching-system.md`, `docs/features/screenshot-import.md` — tactic hierarchy, Apply/Apply All, and screenshot-detection sections rewritten to match; new lessons learned (verify against the real engine via a `vm` sandbox, not a hand-written mirror; verify screenshot reconstructions for connectivity + unique-solution before trusting them; recognize when an input breaks an algorithm's assumptions rather than replaying it anyway).

**Verification**
All tactic-engine changes tested by loading the real `app.js` + `solver.js` into a Node `vm` sandbox and calling `generateHintText()` directly (not a hand-written mirror), against: two hard reference boards reconstructed pixel-by-pixel from real screenshots (levels 840 and 1130, both confirmed unique-solution before use) and a 75-puzzle regression (sizes 5–10). Zero unsound eliminations throughout.
